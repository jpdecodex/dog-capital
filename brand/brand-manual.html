01 — The Mark

A classic serif wordmark — DOG CAPITAL — with a dog's head silhouette worked into the counter of the O. It's the brand's one moment of character; everything else in the system stays quiet so this can carry it.

Primary lockup — full wordmark with dog-in-O

Standalone mark — the dog-in-O alone, cropped out. Use anywhere the full lockup won't fit or read: favicon, app icon, social avatar, small UI chrome.

Never redraw it. Always crop/export from the same master file as the full lockup so the two stay identical.

Clear space & minimum size

  • Clear space: minimum padding on all sides equal to the cap-height of the "D," before any rule, edge, or nav item — the dashed box above.
  • Header size: 36px tall in the site header. Scale proportionally — never stretch on one axis.
  • Minimum size: don't render the full lockup below 28px tall — the "CAPITAL" small-caps line breaks up at that scale. Below that, drop to the standalone mark.
  • Color: single ink black (#1C1C1C) only, on white. No reversed/white version exists yet — don't place the mark on a dark or colored field until one is made.
Production note

The only source file today is a flattened raster with no alpha channel. Every crop here (lockup, mark, favicons) was produced by chroma-keying the background out — workable at header size, but soft at anything larger (a deck, print, a big favicon slot). Get a true vector (SVG/EPS/AI) or a 2000px+ PNG-24 with real transparency from whoever produced it, and regenerate every asset in this manual from that file. Placement and sizing rules above don't change.

03 — Typography

Two voices, deliberately kept apart: a serif mark for identity, a system sans for everything the reader has to work with. Adopting serif headings site-wide would compete with the mark instead of framing it, and would cost legibility in dense metrics tables.

Dog Capital

Serif — mark only. Never used for body copy, headings, or UI.

04 — Voice & Tone

Plain, precise, unhyped. This is a systematic research shop writing for people who read backtest tables for a living — not a fintech consumer app.

Do
  • State the number, then the one line of context it needs.
  • Name what's live vs. paper plainly — no euphemism.
  • Use "we" sparingly; let the data carry the argument.
Don't
  • Superlatives ("industry-leading," "revolutionary").
  • Exclamation points, emoji, or growth-hacker urgency copy.
  • Burying a drawdown number below the fold.

06 — Charts

Every figure is generated in matplotlib, no live charting library. White canvas, no top/right/left spines, horizontal hairline gridlines only, flagship series in accent green, everything else a gray ramp.

Benchmark Strategy
  • Two lines, one comparison, one conclusion — the target shape for every headline equity/drawdown chart.
  • No in-figure title — the page's own H2 carries it.
  • Drawdown charts stay grayscale, no accent — the least appropriate place to reach for color.
  • Font stack: Helvetica Neue, Arial, DejaVu Sans (matplotlib's own fallback, so it never silently drops to Times).

Full palette constants, rcParams module, and per-chart-type specs live in the Visual System Spec doc.

07 — Don't

  • Don't redraw, recolor, or "clean up" the dog-in-O — export it from the master file only.
  • Don't set headings or body copy in the serif — it's mark-only.
  • Don't introduce a second accent color. If a chart needs more than one series distinguished, use the gray ramp before reaching for a new hue.
  • Don't add drop shadows, gradients, or card elevation — the whole system is flat, hairline-bordered.
  • Don't place the mark on a colored or dark background until a reversed version exists.
